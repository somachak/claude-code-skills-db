# Spec-Driven Development

**Category:** ai-productivity · **Priority:** supporting · **Audience:** full-stack, backend

Enforces writing a complete specification before any implementation begins. Use when starting a new feature, API, or subsystem — especially when working with AI coding agents that over-implement. Produces a nine-section spec with RFC 2119 requirements, acceptance criteria, and test stubs before a single line of code is written. Catches scope creep and ambiguity early. Different from designing-workflow-skills (which designs skills) and designing-rest-apis (which designs API contracts).
