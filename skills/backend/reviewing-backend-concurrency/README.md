# Backend Concurrency Review — plain-English guide

## What this skill is for

Reviews concurrent backend code for race conditions, deadlocks, ordering bugs, lock contention, and inconsistent side effects. Use when services coordinate shared state, workers, or high-throughput workloads.

## In simple terms

This skill helps with backend concurrency review work so you do not have to explain the same rules or workflow every time.

## When you would use it

- race condition
- deadlock
- concurrency bug
- parallel worker
- lock contention
