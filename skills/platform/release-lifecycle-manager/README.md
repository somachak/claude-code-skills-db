# Release Lifecycle Manager

**Category:** platform · **Priority:** supporting · **Audience:** full-stack, backend

Manages the full software release lifecycle: semantic versioning, conventional commits, automated changelog generation, release readiness gates, hotfix procedures, and rollback triggers. Use when shipping a versioned release, coordinating a hotfix, writing a changelog, or establishing a release process for a team. Different from running-release-checklists which executes a single pre-release checklist. Covers Git Flow, trunk-based, and GitHub Flow workflows.
