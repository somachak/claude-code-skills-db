# Coverage Gap Reviewer — plain-English guide

## What this skill is for

Reviews what is not tested and prioritizes the highest-risk missing scenarios. Use when shipping new code, after large refactors, or after incidents that revealed blind spots.

## In simple terms

This skill helps with coverage gap reviewer work so you do not have to explain the same rules or workflow every time.

## When you would use it

- coverage gaps
- missing tests
- risk-based testing
- test blind spots
