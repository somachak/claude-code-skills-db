# Data Skills

Schema design, SQL review, migrations, search/indexing, analytics instrumentation, and ETL validation.

## Included skills

- [`designing-database-schemas/`](./designing-database-schemas/) — Designs relational schemas, keys, constraints, and normalization tradeoffs for operational systems.
- [`designing-search-and-indexes/`](./designing-search-and-indexes/) — Designs search schemas, indexing strategies, ranking tradeoffs, and query filters.
- [`implementing-postgres-vector-search/`](./implementing-postgres-vector-search/) — Implements semantic, vector, and hybrid full-text search in PostgreSQL using pgvector and ParadeDB (pg_search).
- [`modeling-analytics-events/`](./modeling-analytics-events/) — Models analytics events, naming conventions, user properties, and event contracts for trustworthy product analysis.
- [`planning-data-migrations/`](./planning-data-migrations/) — Plans safe schema and data migrations with backfills, rollout sequencing, rollback paths, and runtime compatibility checks.
- [`reviewing-sql-performance/`](./reviewing-sql-performance/) — Reviews SQL queries for plan quality, indexing opportunities, cardinality traps, and unnecessary scans.
- [`validating-etl-pipelines/`](./validating-etl-pipelines/) — Validates ETL and ELT pipelines for freshness, schema drift, lineage breaks, duplication, and reconciliation errors.
